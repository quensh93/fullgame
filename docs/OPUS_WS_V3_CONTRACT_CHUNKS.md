# OPUS WS v3 Contract Chunks

Use these chunks one-by-one when the agent cannot read local files.

## Chunk 1/6 - Transport + Envelope
- Endpoint: `/ws-v3`
- Protocol: `v3`
- Required outbound base fields: `type`, `protocolVersion`, `traceId`
- Required gameplay outbound fields: `clientActionId`, `data.stateVersion`
- Canonical inbound metadata: `eventId`, `traceId`, `serverTime`, `protocolVersion`, `stateVersion`, `clientActionId`
- `ACTION_ACK` means accept/dedup, not final apply success.
- `STATE_RESYNC_REQUIRED` => immediate `GET_GAME_STATE_BY_ROOM` => consume `STATE_SNAPSHOT`.

## Chunk 4/6 - Signal Types
Request/response + async signals are fully defined in `signal_payload_by_type`.
Important: mixed casing is intentional.
- Lowercase streams: `room_list`, `room_created`, `room_update`, `room_cancelled`, `room_removed`, `ownership_transferred`
- Uppercase direct signals: `AUTH_SUCCESS`, `ACTION_ACK`, `ROOM_CREATED`, ...
- `GAME_INVITATION_SENT` is dual-shape (response and async push).

## Chunk 6/6 - Error Policy + Normalization
Error policy:
- `AUTH_REQUIRED`: re-auth flow
- `AUTH_EXPIRED`, `TOKEN_REVOKED`, `INVALID_TOKEN`: clear auth + disconnect + login
- `ACTION_REJECTED`: surface and clear correlated pending action
- `STATE_RESYNC_REQUIRED`: snapshot pull and replace state
- `APP_VERSION_UNSUPPORTED`: stop realtime and force update UX
- `RATE_LIMITED`: backoff

Normalization:
- alias mapping for lowercase room streams
- dual-shape normalization for `GAME_INVITATION_SENT`
- `STATE_SNAPSHOT` can be consumed by `GAME_STATE` listeners
